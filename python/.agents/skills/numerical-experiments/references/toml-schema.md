# TOML Schema Notes

## 目标

将 `experiments/*.toml` 作为可复用实验配置的固定位置，用来承载实验定义、参数组织和输出约定。

## 约定

- MUST: 可复用实验配置放在 `experiments/*.toml`。
- SHOULD: TOML 负责表达实验需求、运行入口、参数组织和输出位置。
- SHOULD: 字段按需设计，不强制固定 schema。
- SHOULD: 优先保持字段命名自解释，避免把大量隐式约定藏在脚本里。
- SHOULD: 当实验逐步演化时，优先在原有字段基础上平滑扩展，而不是频繁改换完全不同的组织方式。
- SHOULD: 若实验需要断点续跑，保持 case 标识和参数展开逻辑稳定，避免同一配置在不同运行中生成无法对齐的 case 集合。

## 使用建议

- 将 TOML 视为实验定义的稳定入口，而不是临时备注。
- 将真正影响实验运行和结果解释的参数写入 TOML，不要只留在命令行历史或对话里。
- 当同一实验会重复运行、修改参数或交给后续任务继续使用时，优先补全 TOML，而不是依赖一次性内联参数。
- 若项目中已经存在成熟配置约定，优先与现有结构兼容，再引入本 skill 的默认组织方式。
