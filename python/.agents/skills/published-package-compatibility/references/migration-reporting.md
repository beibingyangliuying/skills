# Migration Reporting

## 目标

在修改后明确验证兼容面，并把影响与过渡方式讲清楚。

## 验证要求

- MUST: 修改前先识别哪些符号、导出模块、命令、配置项或文件格式已经对外暴露。
- MUST: 修改后至少检查以下内容是否仍一致或具备兼容过渡：
  - `README.md` 中的公开用法
  - 示例代码或文档中的 import 与调用方式
  - 测试中的公开接口覆盖
  - 包导出入口与 CLI 入口
- SHOULD: 若任务涉及 `pyproject.toml`、依赖、入口点或构建配置，额外检查发布与安装路径是否仍成立。
- MUST: 若存在潜在 breaking change，必须在结果汇报中明确指出影响面、迁移方式与是否已有兼容层。

## 结果汇报

- MUST: 明确说明识别到了哪些“对外发布包”信号。
- MUST: 明确说明本次修改影响了哪些公开接口边界。
- MUST: 若保留了兼容层、旧别名或过渡方案，汇报中必须说明保留方式。
- MUST: 若无法避免破坏性变化，必须明确提示用户，而不是默认替用户接受 breaking change。
