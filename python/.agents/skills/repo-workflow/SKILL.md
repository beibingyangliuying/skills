# Repo Workflow

## 核心目标

在正确的仓库环境中执行命令，并以最小代价验证修改结果。

## 边界说明

- MUST: 本 skill 只约束环境确认、命令执行与验证流程，不替代其他专项规则。
- SHOULD: 若任务涉及新增或重构测试文件的组织方式与命名方式，优先同时遵循 `python-test-organization`，不要把测试设计约束并入本 skill。
- SHOULD: 若任务只是静态改写代码而不需要运行命令，本 skill 可以只提供最小化的验证判断，不必强行扩展工作流步骤。

## 主流程

### 1. 环境确认

- MUST: 在运行本仓库相关 Python 命令前，先查看仓库已有的环境约定。
- MUST: 若仓库未明确给出虚拟环境激活方式，必须暂停后续 Python 命令、脚本运行与验证工作，并向用户询问应如何激活环境或应使用哪个入口。
- MUST NOT: 不得在缺少仓库依据时，自行尝试猜测式的虚拟环境激活方式或临时发明执行入口。
- SHOULD: 若虚拟环境已经成功激活，运行脚本、模块，以及为用户提供测试命令时，优先使用当前 shell 中可直接执行的 Python 命令，例如 `python -m pytest`、`python -m package.module` 或 `python script.py`。
- MUST: 只有当任务本身属于依赖安装、环境同步、锁文件维护或其他环境管理操作时，才改为参考 `references/uv-active-rules.md` 中的规则执行。
- MUST NOT: 涉及虚拟环境配置、依赖安装、环境同步或解释器环境变更时，不得使用 `pip`。
- SEE: 需要环境选择顺序时，读取 `references/environment-selection.md`。
- SEE: 需要 `uv --active` 的完整约束时，读取 `references/uv-active-rules.md`。

### 2. 测试执行

- MUST: 环境确认完成后，默认提供最相关的最小测试集所对应的可直接执行 PowerShell 命令，并交由用户自行运行。
- MUST: 只有当用户明确要求 AI 执行测试时，才自行运行最相关的测试；否则不要默认代跑测试。
- SHOULD: 默认优先使用 `pytest` 作为提供给用户的测试入口。
- SHOULD: 若虚拟环境已经成功激活，为用户提供 PowerShell 测试命令，或在用户明确要求 AI 执行测试时，优先使用当前 shell 中的 Python 命令，例如 `python -m pytest`，不要默认切到 `uv` 入口。
- SHOULD: 修改公开函数、核心逻辑、数据处理流程或 bug 修复后，优先提供最相关的最小测试集，避免无必要地默认扩大为全量测试。
- SHOULD: 若修改触及已有 doctest 覆盖的函数、模块说明或示例，应尽可能保留现有 doctest，优先同步更新其内容以匹配新行为，而不是直接删除；只有在 doctest 明显失效、重复、或与当前仓库测试策略冲突时，才考虑移除。
- MUST: 若 AI 未实际运行测试，必须明确说明本次提供了哪些测试命令、哪些测试仍待用户执行，以及当前验证缺口是什么。
- MUST: 若 AI 实际运行了测试，必须明确区分哪些结果来自 AI 本次执行，哪些验证仍待用户完成。
- SEE: 需要测试范围、doctest 边界或命令验证细则时，读取 `references/verification-scope.md`。

### 3. 脚本与命令验证

- SHOULD: 先做最小可验证修改，再运行最小必要命令验证。
- SHOULD: 脚本运行与命令验证优先沿用已激活环境中的 shell Python 命令；`uv` 只用于仓库约定或任务本身要求的环境管理操作。
- MUST: 不要假装命令已经成功运行；只报告实际执行和实际结果。
- MUST: 若未实际运行测试或命令验证，必须明确说明原因；若仅提供了测试命令，也要说明仍待用户执行。

## 交付检查清单

在完成任务前，检查：

- 是否已先查看仓库现有环境约定
- 若未找到明确激活方式，是否已暂停执行并向用户询问，而不是继续猜测环境入口
- 跑代码、跑模块，或为用户提供测试命令时，是否优先使用了已激活环境下的 shell Python 命令
- 涉及环境或依赖变更时，是否避免使用 `pip`，并在必须使用 `uv` 时遵循了 `uv-active-rules`
- 若已激活环境，是否为所有实际执行的 `uv` 命令附加了 `--active`
- 是否已在环境确认完成后，为用户提供最相关、可直接执行的最小测试命令
- 若修改触及已有 doctest，是否已尽可能保留并优先更新其内容，而不是无必要删除
- 是否已明确标注哪些测试命令已提供给用户、哪些结果来自 AI 本次执行、哪些验证仍待用户完成
- 是否避免把一般代码改写错误地扩展成工作流任务
