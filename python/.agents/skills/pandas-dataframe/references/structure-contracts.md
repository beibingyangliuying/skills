# Structure Contracts

## 目标

让 DataFrame/Series 的输入输出结构约束在代码和文档里都足够明确。

## 结构约束必须显式

- MUST: 若函数输入或输出为 DataFrame/Series，文档中必须说明关键结构约束。
- MUST: 至少在必要时说明以下内容：
  - 必需列名
  - 索引含义
  - 分组键
  - 结果行粒度
  - 关键缺失值假设
- SHOULD: 对“输入必须已排序”“列必须唯一”“索引必须单调”等前置条件做显式说明。

## 结果表达

- SHOULD: 数据处理函数尽量返回结构稳定、列名清晰的结果。
- SHOULD: 避免返回“靠约定猜测含义”的匿名元组或字段不稳定的对象。
- MUST: 若返回 DataFrame 的列具有业务含义，文档必须写清楚。
- MAY: 对结果列进行适度排序或重排列顺序，以增强可读性，但不要破坏既有接口约定。
