# Matplotlib Figures

## 何时使用

当任务包含以下内容时使用本 skill：

- 编写或修改 Matplotlib 绘图代码
- 保存图像到文件
- 调整图形布局、边距、标签、分辨率或导出格式
- 整理科研绘图脚本

## 核心目标

输出版式稳定、标签清晰、适合科研使用的图形结果，并尽量保持与仓库既有图形风格一致。

## 边界说明

- MUST: 本 skill 只约束 Matplotlib 绘图与导出相关工作，不替代通用数据处理或接口设计规范。
- SHOULD: 若任务重点是 DataFrame/Series 处理而不是出图，优先使用 `pandas-dataframe`；若主要是一般 Python 结构重构，优先遵循 `AGENTS.md` 与其他通用 skill。

## 执行规则

### 1. 布局处理

- SHOULD: 在保存或显示图像前，优先处理布局问题。
- SHOULD: 通常可使用 `plt.tight_layout()`。
- MUST: 若代码已使用 `constrained_layout=True` 或已有明确的手动布局控制，不要机械重复调用 `tight_layout()`。
- MUST: 不要为了套用布局规则而破坏已有图形效果。

### 2. 导出规则

- SHOULD: 使用 `plt.savefig()` 时优先显式设置 `bbox_inches="tight"`。
- SHOULD: 图像导出格式优先遵循仓库既有产出规范。
- SHOULD: 若无明确要求，科研绘图默认优先考虑矢量格式，如 `svg` 或 `pdf`。
- MAY: 在项目明确要求时使用 `format="svg"`、`dpi=600` 等导出参数。
- MUST: 若任务或现有脚本已明确指定导出格式，不擅自改成其他格式。

### 3. 图形可读性

- MUST: 轴标签、标题、图例、刻度应可读且语义明确。
- SHOULD: 尽量避免标签重叠、图例遮挡、边距裁切。
- SHOULD: 修改绘图逻辑时优先保持既有配色、线型与视觉语义，除非任务明确要求调整。

### 4. 数据留存

- MUST: 若任务明确要求科研复现、报告资产沉淀、批量出图流程，或仓库已有同类约定，导出图像时同时保留可复现绘图输入。
- SHOULD: 需要保留绘图输入时，将原始数据保存在图像输出同一个根目录下的 `plot_data/` 子目录中。
- SHOULD: 原始数据优先保存为 `csv` 格式，前提是该格式能无损或合理表达绘图输入数据。
- MAY: 当数据结构不适合直接写成 `csv` 时，改用更保真的格式，如 `json`、`npz` 或其他合适格式；回退格式也应保存在 `plot_data/` 目录中。
- SHOULD: 对普通小型绘图修改，不要机械新增原始数据落盘，除非任务或仓库约定明确要求。
- MUST: 若当前任务需要保留绘图输入，不要仅保存截图、序列化 figure 对象，或只把少量数据常量内嵌在代码里而不落盘。

### 5. 代码组织

- SHOULD: 将数据准备与绘图逻辑适度分离。
- SHOULD: 将数据导出与绘图导出流程协调组织，避免出现图已生成但复现数据未落盘的情况。
- SHOULD: 对重复绘图设置提取为辅助函数或局部配置。
- MUST: 不要为微小绘图任务引入过重抽象。

## 交付检查清单

在完成代码前，检查：

- 布局是否稳定
- 保存参数是否符合任务或仓库要求
- 标签与图例是否清晰
- 是否破坏了既有图形风格
- 导出格式是否与使用场景一致
- 若任务需要保留绘图输入，是否同步保留了原始数据
- 若保留原始数据，是否优先使用 `plot_data/` 目录和合适的持久化格式
